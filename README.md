# ssl-index-pages

## 项目简介

这是一个用于归档和发布独立 HTML 页面的仓库。本仓库本身不针对任何特定域名或网站，仅作为多个静态 HTML 页面的集中存放与展示空间。

通过 GitHub Pages 功能，这些页面可以直接被访问，适合用于测试、演示、存档或轻量级信息展示等场景。

## 目录说明

```
ssl-index-pages/
├── index.html          # 仓库首页/导航页
├── pages/              # 存放各独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
└── README.md           # 本文件
```

- `index.html`：仓库根目录下的默认页面，可作为所有归档页面的导航或总索引。
- `pages/`：存放所有独立 HTML 页面的文件夹，可根据需要添加或整理子目录。
- `README.md`：本说明文件。

## 页面归档说明

本仓库中的所有 HTML 页面均为独立归档，彼此之间无强关联。每个页面可以拥有独立的样式、脚本和内容，适用于以下用途：

- 临时或长期的静态页面存档
- 前端功能或布局的测试样本
- 无需后端支持的轻量信息展示
- 个人或团队内部资料整理

页面命名应尽量简洁明了，避免使用特殊字符。建议在 `pages/` 目录下按主题或时间建立子目录，以保持结构清晰。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，或在根目录直接添加（需注意路径冲突）。
- 更新页面：直接修改对应 HTML 文件，提交后自动通过 GitHub Pages 部署。
- 删除页面：移除对应文件即可，注意检查是否有其他页面引用该文件。
- 建议在每次提交时附上简要的更新说明，便于版本回溯。

## 注意事项

- 本仓库不存储任何敏感信息或受版权保护的内容，所有页面应为公开可访问。
- 页面内容需符合 GitHub 服务条款和适用法律法规。
- 若启用 GitHub Pages，请确保仓库设置为公开或按需配置访问权限。
